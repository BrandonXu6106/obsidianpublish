## 2. 课前预习的学习顺序
### 2.1. 审美提升练习
#### 2.1.1. 日常收集
除了进行软件学习外，要重点提升对我们设计对象（也就是APP和UI界面）的认识，对界面内容、好坏的认识要依靠自己积累，无法通过外部强行灌输。尤其是提升对UI设计的基本审美，能大致判断界面视觉效果的好坏，而不是对这些设计对象感到异常陌生。
对界面案例的认识、积累不足，会导致你们无法判断自己设计成果的好坏，这会导致对自己设计要求的标准过低，进步极慢，相当于没有品尝过蛋糕甜品就开始学习烘焙，你没办法很好的创造自己不了解的东西。

**所以审美提升是前期准备中最重要的工作 （没有之一） 。**

前期提升审美的主要途径就是 —— 看！不用过于纠结背后原理和逻辑，对作品好坏的评判是通过大量观看和不同作品集之间对比形成的。查看UI设计案例的主要渠道：
- 渠道1：最重要的一种，从**应用商店**下载不同**大厂产品**，查看它们的不同板块和下级页面。这些应用代表国内工资最高等级最高的UI设计师的实战产出，也是你们未来工作主要输出对象。
- 渠道2：通过**花瓣、美叶**等平台，查看相关的**画板案例**，但案例平台和上方比起来的问题，就是虽然作品好像收集多且丰富，但很多界面已经非常古老，跟不上UI快速革新的节奏，或者完全是飞机稿你们不一定识别得出来，
	- [花瓣网](https://huaban.com/)
	- [美叶](https://www.meiye.art/)
	- [Pin](https://www.pinterest.com/pin/8373949303104952/)

需要观看的分享：[设计师如何提升审美？](https://mp.weixin.qq.com/s?__biz=MzI0Njc5NzM5OQ==&mid=2247486458&idx=1&sn=9f9656ad752fcc4180c663a9bbf18184&chksm=e9b88ec2decf07d43b59f39c7ec049c473d90ce957a6b22a25db52ef2026a59709befd042203&token=827925191&lang=zh_CN#rd)

### 2.4. 用户体验的深入理解
用户 体验和UI设计关系极大，但是 **理解用户体验必须从自身体验作为起点** ，而不是从书籍、理论、模型中了解。约等于学习美食品鉴，必然要从自己的品尝经验中出发，而不是在没有味觉的时候听别人形容食物的口味。
虽然我们用什么产品都会有自己的用户体验，但最好的体验积累来自 “对比”。即使用控制变量的形式，使用不同的产品（建议工具类产品）来总结它们体验的优劣。

以下举例实践的方式，可以自己替换不同的需求和产品：
1. 确定需求：要记录课程的笔记
2. 选择应用：Notion、飞书、语雀、印象笔记、有道云
3. 开始实践：每个产品做目录并记录3-5篇，或一个章节的笔记
4. 进行总结：
	a. 每个产品的体验打分，10分给几分
	b. 这个产品的优点总结：
		i. 什么功能点打动你，原因是什么
		ii. 什么功能只有它有其它产品没有，或做的没它好
	c. 没产品的缺点总结：
		i. 哪些功能做的不好，原因是什么
		ii. 哪些功能点缺少，不能满足你的需求

体验感受不止是简单对比，而是你真的站在用户角度上去使用，所以需要给自己创建出需求，再去 **正常使用** ，然后再做后续的总结。

而工具产品的使用体验是最直观的，最适合入门。所以不管是选做笔记，还是P照片、背单词、运动、冥想类工具都可以，重要的是自己去用，而不是走马观花随便看看。

## 3. 其他教学课程的学习
### 3.1. HTML CSS 教学
HTML+CSS是UI设计师必学的“编程技能”，入门它们可以更好地理解UI界面的设计逻辑、实现方式，以及在未来工作中完成更好的交付工作。

HTML+CSS 只是简单的脚本计事语言，不是真正的“编程语言”，所以很容易学习，画一个周末的时间看完做个简单的案例，理解它们得相关应用逻辑即可。

## 4. 阅读建议
推荐书籍主要作为闲时读物，用于补充相关知识、理论、认知的方式。要注意不要把它们当成提高设计能力的一环（短期内做不到），因为设计是实践型技能光靠看书弥补不了动手能力的缺失。

但尽可能培养阅读习惯，因为阅读会培养我们深度思考的逻辑能力，对于后续建立产品、交互、体验思维有巨大的帮助。
