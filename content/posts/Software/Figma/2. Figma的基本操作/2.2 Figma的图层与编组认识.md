## 1. Figma的图层与编组认识
Figma Design 的设计过程需要创建和置入**不同的元素**（如文本、图标、图片等），并对它们进行设置实现最终的设计效果。而这些元素，它们都会在左侧资源面板中**映射出一个对应的“图层 Layer”**，且不同的元素类型会用不同的图标标识。

**==理解图层相关知识是 Figma 前期学习过程中最重要的知识点之一== ，因为不同图层会有不同的属性设置和特性，在后续的复杂操作中会遇到的问题90%以上都可以在图层列表中找到答案。**
图层可以分为**一般图层**和**编组图层**两个大类 
![](https://static.uxbaike.com/uploads/2026/01/d2b5ca33bd970f64a6301fa75ae2eb22-29.png)
- **一般图层**：==没有下级==，无法展开，==单一==图层，画布里==不显示名称==
	- 矢量 **vector**
	- 文本 **text**
	- 图片 **image**
	- 切片 **slice**
- **编组图层**：有下级，==可以展开==下级内容，一种==合集==，==frame/component/auto layout 显示名称==
	- **Section** (shift + s)
		- ==最高优先级==，只能放在==最上级==，其余编组不可包含section
		- 下级可以再放其他编组/单一图层
		- section下，还可以再嵌套section
	- **Frame** (ctrl + alt + g)
		- frame下可以嵌套frame
	- **Group** (ctrl + g)
	- **Boolean**
		- 矢量图形的布尔运算
		- 视觉上是一个图层，但下级可展开，矢量可再单独选中
	- **Component**
		- 特殊编组
	- **Auto Layout** 

> [!info] Group vs Frame
> - **Group**
> 	- 定义
> 	- 劣势
> 		- 没有自己的**尺寸**
> 		- 没有自己的**属性**(eg.名字)
> 	- 优势
> 		- 方便**解组**和**调整** (**统一调整**==下级元素==大小颜色等)
> 		- 不影响**其它元素拖拽**
> - **Frame**
> 		- 相当于视图 View
> 		- ==只有内容被置入到 Frame== 中，才能在==演示模式下被显示==出来
> 	- 劣势
> 		- **解组影响排版**
> 		- 只能**调整本身**大小颜色等
> 		- 拖拽其它元素时，**容易拖入frame中**
> 			- ps. 
> 				- ==想清楚放frame的意义目的，否则会拖累设计过程，不是有这个功能就一定要用==
> 				- 新手**在前期实践中，如果找不出创建 Frame 的理由，就尽量用一般编组进行编组即可** ，不要强行使用 Frame 增加理解负担。
> 	- 优势
> 		- 有自己的**尺寸**
> 		- 有自己的**属性**

## 2. 图层的基本操作
- 图层命名   `Ctrl + R` 
- 锁定状态   `Ctrl + Shift + L`
	- 无法用move工具选中，只能手动点图层
- 可见状态   `Ctrl + Shift + H`

## 3. 图层的状态控制
**优先级** ：排序决定==显示==的==优先级==
- 上移一层  `Ctrl +]`
- 移到最上  `Ctrl + Alt + ]`
- 下移一层  `Ctrl + [`
- 移到最下  `Ctrl + Alt + [`

**上下级**：==编组==类图层会形成==上下级==关系
- 创建编组  `Ctrl + G`
- 创建画框  `Ctrl + Alt + G`
- 解除编组/画框  `Ctrl + Backspace

单一图层可以通过**拖拽**的方式放入编组中。
- 我们==必须通过图层面板==才能==确定图层的结构==，无法通过视觉判断。
	- 因为如果frame勾选了clip content功能，无法判断单一图层是在编组里还是编组上层

## 4. 自动布局与组件编组
**自动布局编组** ：
是在创建自动布局时生成的编组形式，有单独的属性设置。
![](https://static.uxbaike.com/uploads/2026/01/d2b5ca33bd970f64a6301fa75ae2eb22-37.png)

**矢量布尔编组** ：
在多个矢量图形进行布尔运算后生成的编组，虽然使用了布尔元算后的新图形，但下级会保留原来的矢量图形，方便进行二次调整。
![](https://static.uxbaike.com/uploads/2026/01/d2b5ca33bd970f64a6301fa75ae2eb22-38.png)

**界面组件编组** ：
Figma 中的组件 Component 应用后的结果。如果直接使用 Group、Frame 创建组件，那么这些基础编组会直接被替换成组件编组，如果只有一个图层创建组件，则它会新建一个组件编组将原有图层置入下级。
![](https://static.uxbaike.com/uploads/2026/01/d2b5ca33bd970f64a6301fa75ae2eb22-39.png)

编组的上下级关系在实践过程中非常重要，错误的关系就无法获得想要的结果，比如：
- 蒙版没有使用编组导致上方所有的图层“消失”
- 元素图层不在Frame内，演示和导出时都“消失”
- 自动布局编组跑到其它编组下级，导致显示错位

所以后续操作如果出现错误，优先从图层开始排查，没有发现错误再检查其它内容。

## 5. 图层列表中的特殊分组
图层列表种还有一种系统级的分组，那就是为元素添加了滚动交互的设置（后面交互章节会说），那么图层就会自动被分为 FIXDE 和 SCROLLS 上下两组。

![](https://static.uxbaike.com/uploads/2026/01/d2b5ca33bd970f64a6301fa75ae2eb22-40.png)

当关闭全部图层的滚动设置后，这个分组也就会自动消失。关闭的方式可以逐一手动关闭，也可以从 FIXED 内把图层拖进 SCROLLS 中（早年不行，后面增加的功能）。

## 6. 图层相关操作的快捷键
- 上移一格： ctrl + \] ，图层在列表中上移一位
- 移到最上：ctrl + alt + \]，图层移动到列表最顶端
- 下移一格： ctrl + \[ ，图层在列表中下移一位
- 移到最下： ctrl + alt + \[ ，图层移动到列表最底端
- 重新命名：ctrl + r，命名需要输入内容后按回车更改
- 锁定图层：ctrl + shift + l，锁定后不可操作
- 隐藏图层：ctrl + shift + h，隐藏后再画布内不可见
- 创建一般编组：ctrl + g
- 创建画框编组：ctrl + alt + g
- 解除一般编组和画框：ctrl + Backspace
