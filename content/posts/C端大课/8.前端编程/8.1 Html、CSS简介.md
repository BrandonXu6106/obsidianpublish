## 1. UI设计师和编程的关系
>招聘信息：需要具备一定的"编程"的能力，html/css
>建筑设计必须学习工程力学个，产品设计必须了解材料学和生产工艺


- 任何设计行业，只懂画图的设计师注定是残缺的，莫得灵魂的，因为**设计是建立在一系列限制下的视觉解决方案**
	- 视觉稿能落地吗?
- 虽然编程语音众多，但我们只需要掌握HTML和CSS即可
	- HTML和CSS是网站开发的**入门**语言，主要主责**页面框架和样式**的制定![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602160149082.png)
	- HTML和CSS它们最大的特点：**简单**
	- 严格意义上来讲，它们都**不是”编程语言”**，因为它们无法完成高级编程语言的条件运算、函数定义等处理。
	- **好处**：
		- 具备**前端开发思路**，了解页面的实现过程
		- 降低和**程序员沟通**的成本，提升协作效率
		- 提高**设计稿落地性**，加快项目执行进程
- 一个常规的网站，主要由客户端(前端)--服务器端(后端)组成![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602160150884.png)
	- 客户端是我们本地加载运行的内容，后端是服务器运行的内容


## 2. HTML的基本认识 
HTML：超文本标记语言，一种对内容进行结构化标记的工具
- 比如我们在一个TXT文档中，输入一段标题一段文本，但这对系统毫无区别：![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602160203064.png)
- 而 HTML 的标签，就是赋予它们对应**身份的铭牌**，方便之后CSS、JS对它进行控制：![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602160203457.png)
	-  这和**Markdown**非常类似，便于客户端区分内容的类型
- HTML 需要创建对应文档才能执行，我们尝试写第一段 HTML 代码：`<h1> Hello World! </h1>`

## 4. 学习的准备工作
展开 HTML 和 CSS 特别容易，只要两样东西:
1. 浏览器：统一用--==chrome==
	- 为什么固定一款浏览器
		1. **不同的浏览器**，有**不同的内核**![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602160234535.png)
			- 浏览器**内核**（Rendering Engine，也叫**渲染引擎**）：浏览器的**核心组件**，负责把你写的 **HTML、CSS、JavaScript** **代码解析并渲染**成用户能看到和**交互的页面**。
			- 浏览器的**内核**拥有对**前端代码的解释权**，而不同的内核对相同的代码解释起来有差异。
		2. 不同浏览器内核，针对**CSS属性的支持**也不同：![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602160220710.png)
	- 为什么选择Chrome：
		1. 简洁大方，打开响应速度快
		2. 开发者调试工具
		3. **Chrome 的 Blink 内核更新快、标准支持全面、工具完善、社区资源丰富**。
		4. 更容易理解代码如何被浏览器解析和渲染，也能避免因兼容性问题而分心。
2. 编辑器
	- 推荐使用：Bracket
