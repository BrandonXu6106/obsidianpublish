## 5. UI设计中的色彩应用思路
### 5.1. UI配色 vs 平面配色 
**平面设计配色方法 ≠ UI设计配色方法**❗❗❗![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602210204250.png)
**平面配色**：==现实世界==展示，干扰因素极多(使用环境)，==意识流==，==融入性==![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602210209560.png)
**UI配色**：==屏幕中==<u>孤立</u>呈现，干扰因素少，==生理性==，==功能性== 

### 5.2. UI配色的学习方法
学习**色彩基础** → 认识**显示逻辑** → 了解**填色对象** → 总结**配色套路**
![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602210213867.png)
#### 5.2.1. 色彩基础
色彩**基本认识**：
- 色彩是人眼接收到的光的不同**波长**
- 色彩通过**折射和直射**的方式**被**我们**感知**——==折射光==/==直射光==![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602210217783.png)

根据色彩**显示方式的差异**，制定了不同的**色彩"描述标准**"：==HSB==、==CMYK==、==RGB==
- **HSB** (不是真实色值，可以==转化为任意色==彩的描述方式)——理论概念——360(色环)×100×100种色
	- **H**：Hue 色相——色彩感知类型
	- **S**：Saturation 饱和度——色彩浓郁程度
	- **B**：Brightness 明度——色彩亮度![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602210228312.png)
- **CMYK** (==印刷==色)
	- **C**：Cyan 青色
	- **M**：Magenta 品红色
	- **Y**：Yellow 黄色
	- **K**：Black 黑色![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602210229047.png)
- **RGB** (==自发光==色)——256×256×256种色
	- Red/Green/Blue![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602210233381.png)


UI设计的**选色逻辑**：先确定==色相==，再确定==明度饱和度==(figma中没有rgb选色)

#### 5.2.2. RGB的光原理
每个**像素点**的背后都有**三个发光晶体管**，通过自发光的**强弱**形成像素的**显示色彩**
**PS**：OLED屏纯黑色不发光![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602210237854.png)
**屏幕**最终显示**色彩**效果，受**多方面影响**
PS：==UI设计==中，"==色准=="==没有设计意义==—你无法控制用户看到的颜色
数字色彩从 **“设计稿”到“用户眼睛”** 之间的整个传输与转译过程：![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602210245692.png)
1. **色彩数据 → 设计师定义** → ==HEX==：`#FF5733`
2. **系统解析 → 色彩配置文件** → ==“数值”==被理解为“==颜色==”
3. **色彩预置 → 厂商预设和策略** →厂商对==色彩==的==主观调校== (显示器预设模式：Standard、Cinema、Game、sRGB 等。)
4. **硬件解析 → 面板物理特性** → 硬件如何驱动==像素发光==
5. **最终显示 → 与最初的定义有偏差** → “色准”在 UI 设计中意义不大——你无法控制用户看到的颜色


- 不同的**设备硬件**不一样，不同**厂商**对硬件呈色的偏好和策略不一样，所以没有所谓的"RGB标准"
	- 颜色配置文件：显示标准，理解为滤镜文件![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602210329476.png)
- RGB只是“约定俗成”，不是绝对标准
- UI设计师无法控制用户看到的“真实色彩”，只能控制相对关系。
- UI设计的核心是“关系”，不是“绝对值”
	- **对比度**：确保文字和背景的区分度，满足 WCAG 可访问性标准。
	- **层次感**：用颜色区分主次信息，而不是追求某个色值的精准还原。
	- **品牌识别**：即使不同设备显示有偏差，用户仍能认出品牌色（例如微信的绿色、Facebook的蓝色）。
- UI设计师不需要追求“绝对色彩”，而是要确保 **在任何设备上都能保持功能性和识别度**。
- UI 设计更强调 **对比度、层次、心理效应**，而不是色准。

### 5.3. 深色模式的认识
是将原设计切换为深色，进行配色的模式!
白色背景下的亮色颜色，在深色背景下需要降低亮度

**深色背景 ≠ 夜间模式**
**深色模式**：背景色是==深色==
**夜间模式**：减少屏幕亮度(==蓝光==)——屏幕变黄

项目中尽量**避免默认深色模式设计**，深色风格的配色思路和浅色完全不同
>tips. 深色模式只是强行把浅色改深
 
 eg. ![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602230034105.png)


配色的最终效果呈现是具有**整体性**的，**配图**的色彩也会**影响**界面的**最终色彩效果**![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602230036377.png)

UI配色的**实践逻辑**
>先能用好纯色，在考虑渐变

先适应**简单**的配色 **→** 再尝试**花哨**的配色：![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602230038325.png)

## 6. UI设计中的配图思路
配图是UI界面设计中最**重要**的**环节**之一，好的配图对界面会起到非常积极的影响
>tips：项目输出的配图都需要经过认真筛选准备 


### 配图的主要原则
画面清晰、主题突出、内容应景、审美优异

- **画面清晰**：图像==质量清晰==，不是"盘包浆"的状态
	- tips. 无需超清原图，可是==直接截图==导入![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602230047329.png)
- **主题突出**：有一个==明确完整的主题物==，而不是繁杂凌乱的

	- eg. ![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602230050123.png)
- **内容应景**：契合==主题==、==环境==、==功能==
	- eg. ![](https://int-1330323257.cos.ap-guangzhou.myqcloud.com//obsidian/pic/202602230053769.png)
- **审美优异**：配图能表现一定的"==高级感=="，能提升页面视觉效果
	- 必须每天审美训练

>tips. **非开发上线**版本配图**没有限制**
